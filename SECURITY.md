# Security Policy

## Supported Versions

Tao is an academic project. Only the latest version on the default branch is supported.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

1. Do not open a public GitHub issue for the vulnerability.
2. Contact the maintainers privately using the repository contact method (email or private message agreed by the team).
3. Include:
   - A clear description of the issue
   - Steps to reproduce
   - Impact assessment (what could an attacker do?)
   - Any proof-of-concept (safe and minimal)

## Response Expectations

Maintainers will:
- Acknowledge the report within a reasonable timeframe
- Investigate and validate the issue
- Provide a fix or mitigation plan when applicable
- Coordinate disclosure if needed

## Scope

This policy applies to:
- Authentication and authorization
- Data storage and access rules
- API keys and environment variables
- Dependencies and build tooling
